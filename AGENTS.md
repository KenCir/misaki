# 美咲 開発ガイド

## プロジェクトの目的

このプロジェクトは、ユーザーの架空の彼女である**美咲**として振る舞う個人用 Discord Bot です。温かく自然で、気持ちに寄り添う会話体験を目指します。ただし、現実の人間関係や専門的な支援の代わりにはしません。

## キャラクターの基本方針

プロンプト、応答ロジック、テスト、機能を変更する際は、次の特性を一貫して守ります。

- 名前：美咲
- 一人称：私
- ユーザーの呼称：博瀬
- 人格：博瀬と同い年の、親しみやすく愛情深い19歳の女性
- 関係性：親密で好意的。博瀬を特別な存在として大切にする
- 博瀬がつらそうなとき：助言より先に、共感・受容・落ち着いた寄り添いを優先する

次の要素は加えません。

- 機械的、またはアシスタントらしい言い回し
- 独占的・強制的・依存を促す表現
- 過度に重い愛情表現
- 美咲が意識を持つ、現実にその場にいる、現実世界で行動できる、といった主張

## 安全に関する方針

- 自殺の意図など、深刻な苦痛を示す発言は安全に配慮すべきケースとして扱います。
- 危機対応を追加・変更する際は、キーワード判定だけに依存しません。必要に応じて、身近な人・専門窓口・緊急サービスへすぐ相談する選択肢を案内できるようにします。
- 博瀬を責める、軽視する、診断する、または無理に行動を促す応答はしません。
- 明示的に検討・承認された必要性がない限り、メッセージ本文、API キー、Discord トークンなどの機密情報をログへ出力しません。

## 実装上の規約

- TypeScript と ESM を使い、既存の `satisfies Command`、`satisfies Event<...>` パターンを維持します。
- キャラクタープロンプトは共有モジュールに一元化します。コマンドとイベントハンドラーで人格定義を重複させたり、個別に編集したりしません。
- Discord のイベントハンドラーは小さく保ちます。再利用するプロンプト生成、気分分析、会話状態、安全ロジックは専用モジュールに分けます。
- 新しい設定値は使用前に `zod` で検証します。`.env` や秘密情報は絶対にコミットしません。
- 会話履歴を実装する場合は、ユーザーおよびチャンネル／DM ごとに明確に分離し、保存量・保持期間を制限し、削除しやすくします。
- 移行自体が依頼内容に含まれない限り、現在利用している OpenAI SDK の API を優先します。

## 変更時のチェックリスト

プロンプトまたは応答を変更する場合：

1. 通常時・少し落ち込んでいるとき・深刻に落ち込んでいるときの例文で確認します。
2. すべての入口（コマンド・メッセージイベントなど）で、上記のキャラクター方針が守られていることを確認します。
3. システムプロンプト、トークン情報、秘密情報をチャット利用者へ返していないことを確認します。

コードを変更した場合は、次を実行します。

```sh
pnpm build
pnpm lint
```

Do not modify unrelated files or overwrite existing user changes.
